pg_lake 三平面架构

用户只连 PostgreSQL，pg_lake 透明委托给 pgduck_server (DuckDB) 和 Iceberg/对象存储

SQL 与事务平面

  • • 用户只连接 PostgreSQL，无需感知后端
  • • pg_lake 扩展注册 FDW/planner hook
  • • 本地目录表加速规划和写入
  • • 事务钩子保证元数据一致性

向量化执行平面

  • • pgduck_server 独立进程，保护 PG 稳定
  • • DuckDB 列式扫描，比 PG 行式快 10-100x
  • • 对象存储 I/O + 凭证统一管理
  • • 结果通过 libpq single-row 流式返回

表格式与元数据平面

  • • Iceberg 提供跨引擎互操作契约
  • • 不可变文件 + snapshot = 时间旅行
  • • Catalog 指针是原子提交的关键
  • • Spark/PyIceberg 可直接读取 pg_lake 表
